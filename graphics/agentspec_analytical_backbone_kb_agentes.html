Camada 5 — stack e padrões (seu brief)

Base React + TypeScript · Vite (recomendado MVP) ou Next.js · Tailwind + Shadcn UI
Dados & formulários TanStack Query · React Hook Form + Zod · Service Layer (sem axios espalhado)
Estado & qualidade Zustand só se sessão/UI global justificar · ESLint/Prettier · Vitest + RTL
Padrão (imagem) Onde no GIFI UI Onde mora a regra
Feature Folder features/auth, features/cenarios, features/dashboard, features/detratores Módulo, não por tipo de arquivo
Container / Presentational Página de simulação vs. charts puros Container coordena; chart só renderiza
Custom Hooks useScenarioUpload, useSimulation, useDetractors Hooks + services — não no JSX
Schema Validation schemas/ espelhando template Camada 1 Zod ↔ contrato de colunas Modo A/B
Service Layer services/simulationApi.ts Único ponto TanStack Query / fetch
State Separation Local (form) · Query (remoto) · URL (modo A/B) · Zustand (sessão mínima) Evitar duplicar preditos no store
Error Boundary Erro de parse/upload e falha de curva Evita tela branca em homologação
Estrutura frontend recomendada (alinhada ao brief) 
src/
  app/                 # bootstrap, providers, ErrorBoundary
  features/
    auth/
    cenarios/          # upload + template Modo A/B
    dashboard/         # curvas TSA / Carga / Extrativos
    detratores/        # consumo Matriz C
    relatorios/        # aderência / desvios (pós-31/08)
  components/ui|layout|shared/
  hooks/ services/ lib/ schemas/ types/ stores/ routes/

Regra crítica: não usar useEffect para orquestrar regra de negócio; TanStack Query para remoto, RHF para form, derivados síncronos para UI.
