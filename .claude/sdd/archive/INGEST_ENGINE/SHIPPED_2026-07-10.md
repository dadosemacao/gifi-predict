## Summary

Implementado o **Ingest Engine** (Camada 2 GIFI): pipeline Python I1–I5 com dual-path batch/online que transforma Excel QM×Processo, fontes TI e uploads de cenário em artefatos L2 versionados (`train_features`, `holdout_features`, `infer_features`, `batch_manifest`) em Parquet+JSON. Validado com o Excel real de 7.573 turnos (7.064 train + 500 holdout), sinais normativos, quarentena, warning matrix e CLI `ingest`.

## Lessons Learned

### Process

- Sprint documental (P0) antes do código evitou retrabalho no contrato de colunas e warning matrix.
- Validação com Excel real deve ser gate obrigatório antes do ship — fixtures sintéticos não cobrem mapeamento PT-BR nem grain datetime.

### Technical

- `business_label_map` no KB precisa ser aplicado em runtime (`build_alias_map`), não só documentado.
- `turno` no Excel está codificado no horário de `Data processo` (0/8/16h) — derivar no connector, não exigir coluna explícita.
- Holdout inelegível deve publicar artefatos mas preservar `current.json` (last-good) — decisão explícita no manifesto.

### Communication

- Judge advisory FAIL em decisões de design (`current.json`) exige documentação no manifesto (`current_pointer_skip_reason`) para evitar confusão operacional.

### Tools & Libraries

- pandas + openpyxl + pyarrow + typer: stack adequada para Marco 1 local.
- pytest `@pytest.mark.slow` para smoke Excel sem penalizar CI rápido.
