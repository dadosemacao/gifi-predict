## Summary

Implementada **auditoria call-by-call** do GIFI Serving: middleware ASGI intercepta 100% dos endpoints `/api/*` (sucesso e erro 4xx/5xx), persiste request/response, métricas, `field_origins` e hash de uploads multipart em **SQLite SSOT** (`logs/serving_audit.db`). Inclui migração versionada, config `audit_enabled`/`audit_db_path` e CLI `scripts/audit_query.py` para consultas operacionais.

## Lessons Learned

### Process

- SDD Brainstorm → Define → Design → Build em um dia funcionou bem para feature transversal (observabilidade) sem impacto na UI.
- Acceptance tests AT-001..AT-007 mapeados 1:1 para pytest acelerou verificação de ship.

### Technical

- **ASGI puro** é preferível a `BaseHTTPMiddleware` para capturar body de request e response sem quebrar multipart.
- **Parser multipart:** `strip(b"\r\n")` no final da section remove bytes válidos do arquivo (newline final do CSV); usar `lstrip` + remover apenas o `\r\n` de boundary.
- **TestClient** exige context manager (`with TestClient(...)`) para rodar lifespan e aplicar migrations no startup.
- INSERT após response completa (trade-off documentado no DESIGN) mantém API responsiva; falha no INSERT não derruba request.

### Communication

- Escopo `(c) todos /api/*` + `(d) SSOT permanente` definido no brainstorm evitou debate sobre JSONL vs SQLite durante build.

### Tools & Libraries

- **sqlite3 stdlib + WAL:** suficiente para MVP (< 10k calls/dia); zero dependência extra.
- **ruff + pytest:** validação rápida pós-build.
